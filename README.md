# 空間 Layout 與物品庫存管理系統

這是一個可直接部署到 GitHub Pages 的靜態網頁版本，用來管理居家空間配置、家具位置、收納容器、物品庫存、標籤分類、提醒與資料備份。

## 主要功能

- 建立套房、雅房等場景，並管理主房間、廁所、陽台、玄關等空間區塊。
- 在 Layout 畫布中拖曳家具，調整大小、旋轉、鎖定、刪除與檢查門後阻擋物。
- 管理門、窗、冷氣、排風扇等牆面設施。
- 進入家具或收納容器內部，記錄桌面、抽屜、層架、箱子等位置中的物品。
- 支援大標籤、小標籤、圖示、數量、有效期限、耗用品預估用完時間與低庫存提醒。
- 家中庫存可依關鍵字、標籤、位置或配置搜尋。
- 支援資源回收桶、更新日誌、資料健康檢查、盤點與補貨清單。
- 支援個人化設定，包括介面風格、語言、字體、字級、版型、顏色與側欄模式。
- 支援新手教學，第一次開啟會自動顯示，也可手動重新播放。
- 支援本機資料同步與 JSON 備份、匯入、載入配置、儲存配置。

## 使用方式

直接開啟 `index.html` 即可使用。

如果瀏覽器對本機檔案權限較嚴格，建議在此資料夾啟動簡易伺服器：

```bash
python -m http.server 8000
```

然後開啟：

```text
http://localhost:8000
```

## GitHub Pages 部署

將本資料夾內容上傳到 GitHub repository 後，可在 GitHub Pages 設定中選擇 `main` branch 的 root 作為發布來源。

建議上傳的檔案包含：

- `index.html`
- `app.js`
- `styles.css`
- `assets/`
- `data/.gitkeep`
- `.nojekyll`
- `.gitignore`
- `README.md`

## 乾淨上傳版說明

本乾淨上傳版只保留程式與必要空資料夾，不包含個人居家資料。

資料資料夾中目前只應該有：

```text
data/.gitkeep
```

請不要把自己的資料檔上傳到公開 repository：

```text
data/layout-inventory-data.json
```

這個檔案可能包含房間配置、家具位置、物品清單、標籤、備註與個人化設定。公開上傳前，請確認它沒有被加入 Git。

## 檔案用途

- `index.html`：網頁入口與主要 HTML 結構。
- `app.js`：Layout、庫存、資料管理、教學與互動邏輯。
- `styles.css`：整體視覺、版面、響應式與個人化樣式。
- `assets/`：圖片、圖示或字體等靜態資源。
- `data/`：保留給本機資料同步使用，乾淨版只放 `.gitkeep`。

## 注意事項

- GitHub Pages 是公開網站，請勿上傳個人資料 JSON。
- 若要保留自己的資料，請使用系統內建備份 JSON 或本機同步資料夾。
- 如果加入自訂字體，建議放在 `assets/fonts/`，再於個人化設定中填入相對路徑。
- 上傳前請再次確認 `data/layout-inventory-data.json` 沒有被加入 Git。
